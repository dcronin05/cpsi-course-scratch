# CPSI-27603 Notes - Project Rules & Workflow Constraints

These are the critical agreed-upon workflow constraints for this repository. They are stored here to enforce strict compliance and must be reviewed regularly.

> [!IMPORTANT]
> **Active Rule Review**: The AI model **MUST** view this rules file at the start of every planning phase (using the `view_file` tool with `IsSkillFile: true`) to ensure all constraints are fresh in context.

---

## 1. Git Versioning & Commit Hygiene
- **Check Status Regularly**: Run `git status` at the beginning and end of every turn to verify branch status and untracked files.
- **Branch Strategy**: Never commit directly to `main`. Create descriptive, kebab-case feature or bug branches (e.g. `feature/name-of-feature`, `bugfix/issue-description`) off `main`.
- **Granular Commits**: Break down changes into small, logically grouped, incremental commits. Commit each step individually rather than combining unrelated edits.
- **Merge Hygiene**: Once a branch is fully completed, merge it into `main`, push to remote, and delete the local feature branch to maintain a clean workspace.

---

## 2. Plan-Driven Development
- **Plan First**: Any complex change, architectural modification, or new feature requires drafting an implementation plan first.
- **Implementation Plan Location**: Draft the plan in the system artifact `implementation_plan.md`.
- **Explicit User Approval**: The AI **must** stop and await explicit approval from the user on the implementation plan before making any edits or executing commands.
- **Task Checklist (`task.md`)**: Track tasks during execution. Mark items as `[/]` (in progress) and `[x]` (completed) as you go.
- **Verification (`walkthrough.md`)**: Record all verification commands, manual checks, and results to guarantee success.

---

## 3. Python Environment (`.venv`)
- **Virtual Environment Isolation**: All Python scripting, executions, package installations, and tools **must** use the local virtual environment `.venv/`.
- **Execution Command**: Never run generic system python commands. Prefix commands with the venv path, e.g. `.venv/bin/python manage_db.py ...` or `.venv/bin/pip install ...`.
- **Dependencies**: Maintain all dependencies in `requirements.txt` and keep the `.venv/` folder ignored in `.gitignore`.
